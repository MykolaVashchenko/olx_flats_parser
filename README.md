# Парсинг квартир з OLX - Висновок
 
## Опис завдання
 
Завдання полягало у зборі оголошень про нерухомість із сайту OLX.ua з використанням бібліотеки Playwright та збереженні структурованих даних у Google Sheets
 
## Що було спарсено
 
Було обрано категорію довгострокової оренди квартир у Києві:

(за бажанням можна змінити місто, змінивши його в кінці URL в коді)
 
```
https://www.olx.ua/uk/nedvizhimost/kvartiry/dolgosrochnaya-arenda-kvartir/kiev/
```
 
По кожному оголошенню збиралися такі дані:
 
| Поле | Опис | Звідки береться |
|---|---|---|
| Назва | Заголовок оголошення | Тег h1 або h4 на сторінці оголошення |
| Ціна | Орендна ціна в гривнях | Тег p[data-testid="ad-price"] на картці |
| Поверх | Поверх квартири | Параметр Поверх: на сторінці оголошення |
| Поверховість | Кількість поверхів у будинку | Параметр Поверховість: на сторінці оголошення |
| Населений пункт | Місто | Тег p[data-testid="location-date"] на картці |
| Площа | Загальна площа квартири | Параметр Загальна площа: на сторінці оголошення |
  
## Як відбувався парсинг
 
**1) Завантаження сторінки каталогу**  
За допомогою Playwright (Chromium у headless-режимі) відкривалася сторінка з переліком оголошень. Щоб уникнути блокування, браузер імітував реальний User-Agent Chrome
 
**2)Збір карток з каталогу**  
Усі картки оголошень відбиралися селектором div[data-cy="l-card"]. З кожної картки витягувалися ціна та посилання на детальну сторінку
 
**3) Фільтрація небажаних оголошень**  
На цьому етапі відсіювалися:
- Квартири на продаж та рекламні оголошення
- Оголошення з аномально низькою ціною

**4) Перехід на сторінку оголошень**  
Для кожного валідного оголошення відкривалася окрема вкладка браузера. Через метод page.evaluate() JavaScript виконувався в DOM сторінки, що дозволило точно знайти потрібні параметри серед тегів та заголовків
 
**5) Збереження в Google Sheets**  
Дані записувалися у Google Таблицю через бібліотеку gspread. Перед записом таблиця очищалася та додавався рядок заголовків
